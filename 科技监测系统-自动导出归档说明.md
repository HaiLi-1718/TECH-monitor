# 科技监测系统 · 新闻归档自动导出说明

本系统会把抓取到的新闻标题永久归档在浏览器本地（IndexedDB：`worldmonitor_news_archive`），
并可通过 Chrome / Edge 的「文件系统访问 API」自动把**新增**标题写入你选择的本地文件夹，
用于长期留存、离线分析或交给下游脚本处理。

**导出范围（白名单）**：仅包含以下四个新闻面板类别——`ai`（AI）、`policy`（政策监管）、
`security`（安全事件）、`biopharma`（生物医药）。`tech`、`startups` 及其它类别会照常归档到
IndexedDB，但**不会**写入自动导出文件夹。完整归档仍可通过「下载 JSON」导出全部类别。

导出始终是**增量**的：每条记录有稳定 id（优先用链接，其次 `来源+标题+发布时间` 的哈希），
已导出的 id 记录在 `export_state` 中全局去重，因此同一条新闻不会被重复导出。

---

## 一、目录结构

从 6.16 起，默认采用「**类别 + 日期**」分层结构，写入到你所选根目录下的 `data/` 子目录：

```
{你选择的根目录}/
└── data/
    ├── ai/
    │   ├── 2026-06-15/
    │   │   └── delta-141407.json
    │   └── 2026-06-16/
    │       └── delta-090322.json
    ├── security/
    │   └── 2026-06-16/
    │       └── delta-090322.json
    └── policy/
        └── 2026-06-16/
            └── delta-090322.json
```

- **类别目录**：使用归档记录的 `category`（FEEDS 的 key）。当前自动导出仅写入
  `ai`、`policy`、`security`、`biopharma` 四个类别；其它类别（如 `tech`、`startups`）不会出现在
  `data/` 下。类别名会做路径片段净化（仅保留 `A-Z a-z 0-9 . _ -`，其余替换为 `_`）。
- **日期目录**：`YYYY-MM-DD`，时区默认 **Asia/Shanghai**（可在设置中修改时区与日期依据）。
- **文件名**：`delta-HHMMSS.json`，其中 `HHMMSS` 为本次导出时刻（同样按所选时区）。
- 一次导出按 `(类别, 日期)` 分组，**每组各写一个文件**；某组 0 条则**不写文件、不建空目录**。
- 全部分组写入成功后，才一次性把这批 id 标记为已导出（任一组写入失败则整批留待下次重试）。

### 文件内容

每个 `delta-*.json` 的结构与旧版一致：

```json
{
  "exportedAt": "2026-06-16T01:03:22.118Z",
  "count": 12,
  "articles": [
    {
      "id": "link:11h2857chp0",
      "category": "ai",
      "source": "AI News",
      "title": "……",
      "link": "https://……",
      "pubDate": 1781071470920,
      "archivedAt": 1781071471118,
      "isAlert": false
    }
  ]
}
```

## 二、设置项

设置入口：右上角 **设置 → 数据与社区 → 定时导出到文件夹**。

| 设置 | localStorage key | 取值 | 默认 | 说明 |
| --- | --- | --- | --- | --- |
| 是否启用自动导出 | `wm-news-auto-export` | `true` / `false` | `false` | 页面保持打开时按间隔执行 |
| 导出间隔 | `wm-news-auto-export-interval-min` | 15 / 30 / 60 / 120 / 360 / 1440（分钟） | 60 | |
| **文件夹结构** | `wm-news-export-layout` | `category-date` / `flat` | `category-date` | `flat` 保持旧行为：在根目录写单个 `news-delta-*.json` |
| **日期文件夹依据** | `wm-news-export-date-basis` | `pubDate` / `archivedAt` | `pubDate` | 按发布时间或归档时间划分日期目录 |
| **时区** | `wm-news-export-timezone` | IANA 时区名 | `Asia/Shanghai` | 决定日期目录与文件名时间；UI 未提供输入框，可在浏览器控制台手动设置 |

> 文件夹结构与日期依据可直接在设置面板的下拉框切换；时区为高级项，默认上海时间已满足绝大多数场景。
> 切换设置只影响**下一次**导出，不会重新整理已导出的历史文件。

### 兼容模式（flat）

把「文件夹结构」切到 **扁平（单个 delta 文件）**，即恢复旧版行为：
所有未导出文章排序后写成一个 `news-delta-YYYYMMDD-HHMMSS.json`，直接放在所选根目录下，不创建 `data/` 分层。

## 三、测试步骤（Chrome / Edge）

1. 用 **Chrome 或 Edge** 打开应用（Firefox / Safari 不支持文件系统写入，会提示不支持）。
2. 进入 **设置 → 智能(Intelligence)**，打开 **新闻归档（News archive）** 开关，让系统开始归档抓取到的标题。
3. 进入 **设置 → 数据与社区 → 定时导出到文件夹**：
   - 点击 **选择导出文件夹**，在弹窗中选一个空目录（如桌面上新建的 `tech-export`），并在浏览器授权读写。
   - 确认「文件夹结构」为 **按类别 / 日期**，「日期文件夹依据」按需选择（默认发布日期）。
   - 打开 **自动导出到文件夹** 开关。
4. 等待应用刷新几轮新闻（或手动切换/刷新各类别面板），让归档库里积累一些标题。
5. 回到设置面板，点击 **立即导出新增**。
6. 到所选文件夹检查目录树，应出现：
   ```
   tech-export/data/{类别}/{YYYY-MM-DD}/delta-HHMMSS.json
   ```
   并且设置面板底部状态会显示「上次导出：新增 N 条，共 M 个文件」（单文件时显示具体文件名）。
7. **去重验证**：再次点击 **立即导出新增**。若期间无新增标题，应提示「没有新增条目需要导出」，且不产生新文件；
   有新增时只会写入新增部分，已导出的旧条目不会重复。
8. **兼容验证**（可选）：把「文件夹结构」切到 **扁平**，再点 **立即导出新增**，确认根目录下生成单个 `news-delta-*.json`。

## 四、相关文件

- `src/services/news-archive.ts` —— 归档读写、增量去重、分层导出（`EXPORT_PANEL_CATEGORIES` 白名单、`exportUnexportedArticlesToDirectory`、`resolveExportPath`、时区日期/时间格式化）。
- `src/services/news-archive-auto-export.ts` —— 定时调度、导出相关 localStorage 设置（结构 / 日期依据 / 时区）。
- `src/services/preferences-content.ts` —— 设置 UI（结构、日期依据下拉框与状态展示）。
- `src/types/file-system-access.d.ts` —— 文件系统访问 API 类型（含 `getDirectoryHandle`）。
- `src/locales/zh.json` · `src/locales/en.json` —— 文案。
- `src/App.ts` —— 启动时调用 `startNewsArchiveAutoExportScheduler()`。
